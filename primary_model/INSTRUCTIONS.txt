# SSGA Meta Labeling - Primary Model

This project implements the foundational backtesting and signal generation logic for the Primary Strategy variant 1, alongside a standardized validation suite for walk-forward limits and cost sensitivity.

## Prerequisites
- Python 3.9+ (3.10+ recommended)
- A virtual environment is highly recommended

## Installation
1. Create and activate a virtual environment:
   python -m venv .venv
   source .venv/bin/activate  # (On Windows: .venv\Scripts\activate)
2. Install dependencies:
   pip install -r requirements.txt
   pip install -r requirements-dev.txt  # If you plan to run tests

## Data Setup
1. Ensure the `artifacts/data/raw/` directory exists.
2. Place your raw input data files (`spx.xlsx`, `treasury_10y.xlsx`, `bcom.xlsx`, `corp_bonds.xlsx`) inside `artifacts/data/raw/`.

## Running the Project
Use the canonical CLI entrypoint (`python cli.py <command>`). Artifacts (CSVs, HTML reports, and plots) are written automatically to `artifacts/reports/`.  
`scripts/run_*.py` wrappers remain available for backward compatibility.

Recommended first command (End-to-end validation suite):
   python cli.py run-validation-suite

Canonical operator command (full reproducibility path):
   make full-validate

Other standalone execution modes:
1. Fast Smoke Test (PrimaryV1 Pipeline):
   python cli.py run-primary-v1
2. Robustness Check (Grid Search):
   python cli.py run-robustness
3. Walk-Forward Check:
   python cli.py run-walk-forward
4. Benchmark Generation:
   python cli.py run-benchmarks

Configs are managed in `configs/experiments/`. You can override paths and thresholds there.

## Running Tests
To ensure the mathematical engine and pipeline architecture is stable, run:
   python -m pytest -q

## Resetting
To reset the project to a fresh state (wipe all artifacts but keep data):
   rm -rf artifacts/reports/*
   rm -rf artifacts/data/clean/*
   (Then recreate required `.gitkeep` placeholders if needed)

## Troubleshooting
- Missing Data: Ensure `artifacts/data/raw/` contains the necessary data files.
- Import Errors: Run commands from the `primary_model/` root directory (the entrypoints bootstrap imports automatically; no `PYTHONPATH` export is needed).

## What Teammates Can Ignore
To keep things simple day-to-day:

Safe to ignore unless developing deeply:

- `.github/` (CI pipelines)
- `Makefile`, `pyproject.toml`, `requirements*.txt` (Setup dependencies)
