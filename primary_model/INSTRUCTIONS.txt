PRIMARY MODEL - SIMPLE RUN GUIDE
================================

This project can be run in 3 simple ways:

1) Run only the main project
2) Run only the isolated test/validation workflow
3) Run both (main project first, then isolated test workflow)


SETUP (DO THIS ONCE)
--------------------
1. Create virtual environment:
   python -m venv .venv

2. Activate it:
   source .venv/bin/activate

3. Install dependencies:
   pip install -e .[dev]


OPTION A - RUN THE MAIN PROJECT ONLY
------------------------------------
Single command:
python scripts/run_modes.py --mode project

What it runs:
- python cli.py run-all

Where outputs go:
- reports/

Equivalent Make command:
- make run-mode-project


OPTION B - RUN TESTS/VALIDATION ONLY (SEPARATE, ISOLATED)
----------------------------------------------------------
Single command:
python scripts/run_modes.py --mode test

What it runs:
- python scripts/run_validation_suite.py --root test --clean-root

This includes:
- pytest
- full pipeline in isolated root
- robustness grid
- strict walk-forward run
- reproducibility logs + manifest

Where outputs go:
- test/reports/

Equivalent Make command:
- make run-mode-test


OPTION C - RUN BOTH (PROJECT + TEST) IN ONE COMMAND
----------------------------------------------------
Single command:
python scripts/run_modes.py --mode both

Order:
1. Main project run (reports/)
2. Isolated test/validation run (test/reports/)

Equivalent Make command:
- make run-mode-both


USEFUL NOTES
------------
- Main project outputs and test outputs are separated by design.
- Main project data/output path: data/, reports/
- Isolated validation path: test/data/, test/reports/
- If you rerun test mode, previous isolated artifacts are replaced because
  --clean-root is applied.
