03 — Data Correction

Treasury Fix

treasury_10y stores yield levels (range: 0.53 → 6.94), not bond prices. Using raw percentage changes as returns inflates volatility 4× and misrepresents bond behavior.

The fix keeps yield levels as-is for reference, but replaces Return with a duration-based total-return proxy:

∑
Duration-Based Total Return Formula

y = yield_percent / 100
price_return = −duration × Δy
carry = y.shift(1) / 12
total_return = price_return + carry

⚠
Assumption: duration = 8.5

Fixed duration typical for a 10Y Treasury. Stabilizes risk behavior but remains an approximation — duration sensitivity analysis is flagged as a future step.

04 — Simulation Engine

Backtest Engine (No-Lookahead)

  • t+1 alignment: weights at date t applied to returns at t+1 via returns.shift(-1). No future peeking.
  • Long-only normalization: negative weights clipped to 0; rows normalized to sum to 1.
  • Turnover: 0.5 × Σ|w_t − w_{t−1}| per period.
  • Transaction cost model: cost = turnover × (tcost_bps / 10000).
  • Current runs: tcost_bps = 0.0 — zero-cost assumption (to be relaxed).
05 — Comparison Baselines

Benchmarks Implemented

  • EqualWeight25: 25% to each of the four assets every period. Static.
  • BuyHoldSPX: 100% SPX static allocation. Pure equity exposure.
  • 60/40: 60% SPX + 40% Treasury. Classic risk-balanced allocation.
  • SimpleTrend: SPX fast/slow SMA regime switching — risk-on (SPX) or risk-off (Treasury).

06A — Mathematical Foundation

Math & Assumptions

Core equations used in the primary-model pipeline, matching the implementation in src/primary_model/.

EQ 1 Treasury Return Proxy — Duration + Carry
\[ y_t = \frac{Y_t}{100}, \qquad r_t^{\mathrm{UST}} = -D\,(y_t - y_{t-1}) + \frac{y_{t-1}}{12}, \qquad D = 8.5 \]
▸ D = 8.5 is a fixed-duration approximation for the 10Y Treasury. Carry = prior month's yield / 12.
EQ 2 Expanding Z-Score (no lookahead)
\[ z_{i,t} = \frac{x_{i,t} - \mu_{i,\,1:t-1}}{\sigma_{i,\,1:t-1}} \]
▸ Normalizes each indicator using only history before time t.
EQ 3 Composite Score
\[ \mathrm{score}_t = \frac{\displaystyle\sum_i w_i\, z_{i,t}}{\displaystyle\sum_i |w_i| \cdot \mathbf{1}_{\{z_{i,t}\,\mathrm{valid}\}}} \]
▸ Weighted average of valid z-scores, normalized by the sum of active weights.
EQ 4 Signal Generation Rule
\[ \mathrm{signal}_t = \begin{cases} \mathrm{BUY}, & \mathrm{score}_t > 0.5 \\ \mathrm{SELL}, & \mathrm{score}_t < -0.5 \\ \mathrm{HOLD}, & \text{otherwise} \end{cases} \]
▸ HOLD carries previous period weights unchanged, minimizing unnecessary turnover.
EQ 5 Backtest — Gross & Net Return
\[ \mathrm{gross}_t = \mathbf{w}_t^\top \mathbf{r}_{t+1} \] \[ \mathrm{net}_t = \mathrm{gross}_t - \mathrm{turnover}_t \cdot \frac{\mathrm{tcost}_{\mathrm{bps}}}{10000} \]
▸ Weights at t are applied to returns at t+1 (no lookahead).
EQ 6 Turnover
\[ \mathrm{turnover}_t = \frac{1}{2} \sum_j \left| w_{t,j} - w_{t-1,j} \right| \]
▸ Half the sum of absolute weight changes. Currently tcost_bps = 0.
EQ 7 Performance Metrics
Annualized Return
\[ \mathrm{AnnReturn} = \left(\prod_{t=1}^{N}(1+r_t)\right)^{\!12/N} - 1 \]
Annualized Volatility
\[ \mathrm{AnnVol} = \sigma(r_t)\,\sqrt{12} \]
Sharpe Ratio
\[ \mathrm{Sharpe} = \frac{\mu(r_t - r_{f,t})}{\sigma(r_t - r_{f,t})}\,\sqrt{12} \]
Maximum Drawdown
\[ \mathrm{MaxDD} = \min_t\!\left(\frac{E_t}{\max_{s \le t} E_s} - 1\right) \]
▸ All metrics annualized over 12 monthly periods. rf = risk-free rate.

10 — Future Work

Limitations / Next Steps

  • Transaction costs: Currently modeled at 0 bps. Add robustness runs at realistic non-zero cost assumptions before production use.
  • Duration sensitivity: Treasury duration is fixed at 8.5. Run sensitivity analysis across a range of duration assumptions.
  • Walk-forward testing: PrimaryV1 uses fixed windows and thresholds. Stability tests required before meta-labeling rollout.
  • Model card: Add a machine-readable JSON model card for downstream automation and governance.
