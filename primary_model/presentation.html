PrimaryV1 in the Meta-Labeling Pipeline

Executive Summary

This report focuses on the primary model only: the first decision layer that converts macro and cross-asset signals into BUY/HOLD/SELL states. In the broader meta-labeling project, PrimaryV1 is the regime engine that defines when risk should be carried before any secondary filtering layer is applied.

  • Primary objective: produce stable market-regime labels from spx_trend, bcom_trend, credit_vs_rates, and risk_breadth.
  • Project role: provide the base directional signal that a later meta-labeling phase can accept, reject, or size.
  • Outputs used here: backtest returns, signal history, weights, benchmark comparisons, and final charts from project_results/reports/.
  • Interpretation: PrimaryV1 favors risk-adjusted consistency over maximum raw return relative to benchmark alternatives.

02 — Comparative Baselines

Benchmarks Section

Benchmarking is used here as a diagnostic lens: each baseline isolates a simpler way to hold risk so we can see what PrimaryV1 adds beyond passive exposure. For a meta-labeling project, this matters because the primary layer should contribute signal quality, not just mirror a common allocation.

The four baselines in project_results/reports/benchmarks_summary.csv cover fixed mix, pure equity beta, classic balanced allocation, and a low-complexity trend timer. Together they create a practical reference frame for risk, turnover, and regime behavior.

Benchmark Definition Project Rationale Risk/Exposure Controlled
EqualWeight25 25% in each asset every month with no timing. Reference for a neutral, mechanical allocator using the same asset menu. Separates stock-selection universe effect from timing logic.
BuyHoldSPX 100% SPX, continuously invested. Pure equity beta benchmark to compare against broad market drift. Anchors performance to directional equity exposure and crash sensitivity.
60/40 60% SPX + 40% Treasury, static rebalance. Institutional baseline for balanced growth/defense trade-off. Controls for classic diversification benefits without tactical overlays.
SimpleTrend SPX trend rule; shift toward Treasury in risk-off regimes. Low-complexity tactical comparator closest in spirit to regime switching. Tests whether PrimaryV1 adds value beyond a single trend switch rule.
ℹ
How to read this benchmark set

If PrimaryV1 beats only BuyHoldSPX, it may just be a defensive profile. If it also competes with 60/40 and SimpleTrend on Sharpe and drawdown, the regime signal is more credible as a reusable primary layer for M2.

03 — Indicators Research

Indicators Research

PrimaryV1 backtested stack: spx_trend, bcom_trend, credit_vs_rates, and risk_breadth, aggregated into composite_score and mapped to BUY/HOLD/SELL states. The design intent is to combine price trend, cross-asset confirmation, and risk-appetite checks in one regime score.

Only PrimaryV1 has complete exported backtest artifacts in the final run outputs. Alternatives below are explicitly conceptual and included to document research direction for the next iteration, not to claim tested performance.

Alternative Indicator Concepts (Conceptual)

Alternative Conceptual Design Status Notes
Alt A: Momentum-Only Use trend factors only (spx_trend + bcom_trend) and drop credit/breadth filters. Conceptual Would test whether signal simplicity improves stability or just increases beta noise.
Alt B: Credit-Led Regime Use credit_vs_rates as a hard gate, then allow trend signals only when credit is supportive. Conceptual Would emphasize stress detection and likely trade less frequently in mixed macro conditions.
Alt C: Breadth + Vol Filter Keep breadth as main state variable and add realized-volatility filter before final label mapping. Conceptual Would target smoother drawdowns at the cost of possible late re-entry after selloffs.
✓
Winner Based on Available Metrics

Based on available reported metrics, PrimaryV1 is the selected winner for the primary layer: highest Sharpe (0.838) and lowest max drawdown (-22.08%) among implemented strategies, with acceptable turnover (0.189) for monthly decisioning.

06 — Conclusion

Conclusion

  • Backtest summary: PrimaryV1 delivered 6.33% annualized return at 7.70% annualized volatility, Sharpe 0.838, max drawdown -22.08%, ending net equity 6.240x.
  • Comparative takeaway: PrimaryV1 did not maximize absolute return, but led on risk-adjusted quality (Sharpe) and downside containment (drawdown).
  • Signal behavior: Win rate 65.08% over 358 evaluated return periods, with BUY/HOLD/SELL counts 197/15/148.
  • M2 implication: prioritize robustness work (non-zero transaction costs, duration sensitivity, walk-forward stability, threshold stress testing) before live deployment logic.
