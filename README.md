# SSGA Meta-Labeling for Tactical Asset Allocation

This repository implements a **meta-labeling** framework for Tactical Asset Allocation (TAA), developed for the Brandeis Field Project with State Street Global Advisors (SSGA).

The core idea is "modeling the model": a secondary ML classifier filters false positives from a primary trend-following signal to improve risk-adjusted performance.

## Project Overview
The system is a two-stage decision engine:
1. **Primary model (strategy signal):** 12-month S&P 500 momentum. A Buy signal is issued when momentum is positive.
2. **Secondary model (filter):** A Random Forest classifier predicts the probability that the primary signal will be successful. Trades are executed only when confidence exceeds a threshold.

## Repository Structure
- `data/`:
  - Raw Excel inputs (expected: `bcom.xlsx`, `spx.xlsx`, `treasury_10y.xlsx`, `corp_bonds.xlsx`).
- `src/`:
  - `data_loader.py`: Loads and merges the four datasets into a time-aligned DataFrame.
  - `features.py`: Generates the five SSGA signals (Momentum, Value, Carry, Volatility, Trend).
  - `evaluation.py`: Unbiased static and walk-forward evaluation engine with threshold selection and metrics.
- `scripts/`:
  - `run_evaluation.py`: CLI runner for static/walk-forward evaluation and report generation.
- `docs/`:
  - `file_relationships.md`: File-by-file technical documentation and dependency map.
  - `sequence_diagrams.md`: Sequence diagrams for static and walk-forward execution paths.
  - `runbook.md`: Operational runbook with commands, outputs, tests, and troubleshooting.
- `notebooks/`:
  - `01_signals_analysis.ipynb`: Main exploratory analysis and backtest workflow (not for final unbiased model/threshold selection claims).
  - `02_unbiased_evaluation.ipynb`: Reproducible evaluation notebook comparing static split vs walk-forward retraining.
- `reports/`:
  - Generated JSON/CSV evaluation outputs from script and notebook runs.
- `images/`:
  - Figures and charts used in reporting.

## Key Functions
- `src/data_loader.py`:
  - `load_data()`: Reads Excel files from `data/`, handles Bloomberg header offsets, cleans columns, and merges all series on `Date`.
- `src/features.py`:
  - `create_indicators(df)`: Computes the five signals and returns a cleaned DataFrame ready for modeling.

## Summary Results (from the project report)
- The secondary model identifies high-volatility regimes (e.g., 2008 and early 2020) where the primary momentum signal tends to fail.
- Out-of-sample performance (2020 to 2025) shows the optimized strategy matching the naive strategy's final value (approximately 1.95) while providing stronger risk control.
- Volatility (`Z5_Vol`) is the most important feature, acting as a reliable risk-off switch during market stress.

Note: Results are sensitive to data revisions, feature engineering choices, and threshold settings. Re-run the notebook if you update inputs.

## Workflow
1. Load and align the data from `data/`.
2. Generate features with `create_indicators`.
3. Train the secondary model and compute prediction confidence.
4. Backtest the naive vs. optimized strategy.
5. Sweep confidence thresholds to evaluate sensitivity.

## Reproducible Evaluation Script
The repository now includes a reproducible evaluation pipeline that:
- Uses chronological train/validation/test splits (no shuffling).
- Selects the confidence threshold on validation only.
- Evaluates final performance on a held-out test set.
- Supports an optional walk-forward mode with rolling retraining and rolling threshold calibration.
- Supports transaction costs (basis points) applied to each position change.
- Supports probability calibration (`none`, `sigmoid`, `isotonic`) for the meta-model.
- Supports utility-based decisioning (expected return minus cost/uncertainty penalty) as an alternative to threshold gating.

Run it from the project root:
```bash
python scripts/run_evaluation.py
```

Artifacts generated by default:
- Static mode (`--mode static`): `reports/evaluation_report_static.json` and `reports/test_trade_log_static.csv`.
- Walk-forward mode (`--mode walk_forward`): `reports/evaluation_report_walk_forward.json` and `reports/test_trade_log_walk_forward.csv`.

Useful options:
```bash
python scripts/run_evaluation.py \
  --mode static \
  --train-frac 0.6 \
  --val-frac 0.2 \
  --decision-policy threshold \
  --probability-calibration sigmoid \
  --objective sharpe \
  --threshold-min 0.30 \
  --threshold-max 0.70 \
  --threshold-step 0.01 \
  --transaction-cost-bps 0.0
```

Utility policy example:
```bash
python scripts/run_evaluation.py \
  --mode static \
  --decision-policy utility \
  --probability-calibration sigmoid \
  --utility-margin 0.0 \
  --utility-risk-aversion 0.25 \
  --transaction-cost-bps 5
```

Walk-forward mode:
```bash
python scripts/run_evaluation.py \
  --mode walk_forward \
  --decision-policy threshold \
  --probability-calibration none \
  --validation-window 60 \
  --min-train-window 120 \
  --objective sharpe \
  --transaction-cost-bps 5
```
